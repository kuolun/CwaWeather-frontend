# 後端 API 修改需求

## 背景說明
前端天氣應用「森森天氣」目前已經將顯示地點改為「新竹縣」，但需要後端 API 支援新竹縣的天氣資料查詢。

## 目前狀態
- **前端 API URL**：`https://weather-check-vibe.zeabur.app/api/weather/hsinchu-county`
- **目前支援的地點**：只有 `kaohsiung`（高雄）
- **需要新增支援**：新竹縣

## 需求內容

### 1. 新增新竹縣的天氣 API endpoint

請在後端新增支援新竹縣天氣資料的 API endpoint。建議的路徑參數為：
- `/api/weather/hsinchu-county` （建議，與前端一致）
- 或 `/api/weather/hsinchu`（如果後端已有此命名規則）

### 2. API 回應格式要求

請確保新竹縣的 API 回應格式與現有的高雄 API 一致：

```json
{
  "success": true,
  "data": {
    "forecasts": [
      {
        "startTime": "2025-01-XX XX:XX:XX",
        "weather": "多雲時晴",
        "minTemp": "18",
        "maxTemp": "25",
        "rain": "20"
      },
      // ... 更多預報資料
    ]
  }
}
```

### 3. 地點參數命名建議

**優先順序：**
1. `hsinchu-county` - 與前端目前設定一致（推薦）
2. `hsinchu` - 如果後端有簡化命名規則

**請確認後端使用的參數名稱，並告知前端是否需要調整。**

### 4. 資料來源確認

請確認：
- 後端使用哪個天氣資料來源（中央氣象署 API？）
- 新竹縣的地區代碼或識別碼是什麼？
- 資料更新頻率是否與高雄一致？

## 測試建議

修改完成後，請測試以下 endpoint 是否能正常回應：
- `GET https://weather-check-vibe.zeabur.app/api/weather/hsinchu-county`

預期回應應該包含：
- `success: true`
- `data.forecasts` 陣列至少有 1 筆資料
- 每筆 forecast 包含：`startTime`, `weather`, `minTemp`, `maxTemp`, `rain`

## 聯絡事項

如果後端使用不同的參數命名（例如：`hsinchu`、`hsinchu_county`、`新竹縣`），請告知前端需要調整的參數名稱。
